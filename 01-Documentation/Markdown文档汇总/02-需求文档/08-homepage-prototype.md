## 1. 设计目标

首页不是功能导航页，而是**快速发起求助页**。

长辈用户打开首页后，应该在 3 秒内理解三件事：
- 这里是帮我解决手机操作问题的
- 我可以直接说话，也可以发截图
- 如果我怕出错，可以联系家里人

---

## 2. 信息架构

### 2.1 首页一级结构

```text
首页
├── 顶部状态区
├── 核心求助区
├── 快捷入口区
├── 最近任务区
├── 常用场景区
└── 安全提醒区
```

### 2.2 模块优先级

| 模块 | 优先级 | 是否首屏可见 | 说明 |
|------|--------|--------------|------|
| 核心求助区 | P0 | 是 | 首页最重要模块 |
| 快捷入口区 | P0 | 是 | 截图、联系家人、常用任务 |
| 最近任务区 | P1 | 是 | 帮用户继续上一次任务 |
| 常用场景区 | P1 | 是 | 降低表达成本 |
| 安全提醒区 | P1 | 是 | 建立信任与风险意识 |
| 顶部状态区 | P2 | 是 | 承载轻量状态信息 |

## 3. 页面原型

### 3.1 首页线框图

```text
┌──────────────────────────────────────────────┐
│ 头像  早上好，王阿姨                设置      │
│ 今天也可以问我：挂号 / 缴费 / 查医保          │
├──────────────────────────────────────────────┤
│                                              │
│              🎤 按住说话                      │
│        不会操作时，直接说出来                 │
│                                              │
│      示例：帮我挂号 / 这个点哪里 / 查医保      │
│                                              │
├──────────────────────────────────────────────┤
│ [上传截图]   [联系家人]   [常用任务]          │
├──────────────────────────────────────────────┤
│ 最近任务                                      │
│ ┌──────────────────────────────────────────┐ │
│ │ 医院挂号                                  │ │
│ │ 进行到第 2 步：选择医院                   │ │
│ │ [继续办理] [查看步骤]                     │ │
│ └──────────────────────────────────────────┘ │
├──────────────────────────────────────────────┤
│ 常用场景                                      │
│ [挂医院号] [查医保] [交水电费] [查社保]       │
├──────────────────────────────────────────────┤
│ 安全提醒                                      │
│ 不认识的链接、验证码、转账要求，先别操作      │
│ [查看提醒]                                    │
└──────────────────────────────────────────────┘
```

### 3.2 首页交互原则

- 首屏只保留一个主操作：`按住说话`
- 次级操作不超过 3 个：`上传截图`、`联系家人`、`常用任务`
- 最近任务优先于推荐内容，帮助用户继续未完成流程
- 所有文案必须口语化，不出现“智能分析”“多模态理解”等技术词

## 4. 模块详细说明

### 4.1 顶部状态区

#### 4.1.1 目标
- 传递欢迎感与熟悉感
- 轻量提示今日可问内容

#### 4.1.2 推荐文案
- `早上好，王阿姨`
- `今天也可以问我：挂号 / 缴费 / 查医保`

#### 4.1.3 设计要求
- 不放复杂通知流
- 不放高密度图标
- 设置入口保持弱化，但可触达

### 4.2 核心求助区

#### 4.2.1 目标
- 用户一眼知道怎么开始

#### 4.2.2 主按钮文案
- 默认：`按住说话`
- 录音中：`正在听你说`
- 结束后：`我来帮你看`

#### 4.2.3 辅助文案
- `不会操作时，直接说出来`
- `也可以发截图给我看`

#### 4.2.4 示例问题
- `帮我挂医院的号`
- `这个页面下一步点哪里`
- `我想查医保余额`

### 4.3 快捷入口区

#### 4.3.1 上传截图
- 按钮文案：`上传截图`
- 说明文案：`看不懂页面时发给我`

#### 4.3.2 联系家人
- 按钮文案：`联系家人`
- 说明文案：`让家里人一起看看`

#### 4.3.3 常用任务
- 按钮文案：`常用任务`
- 说明文案：`直接进入常见问题`

### 4.4 最近任务区

#### 4.4.1 目标
- 避免用户每次从头开始

#### 4.4.2 内容结构
- 任务名称
- 当前进度
- 最近更新时间
- 主按钮 `继续办理`
- 次按钮 `查看步骤`

#### 4.4.3 空状态文案
- `你还没有最近任务`
- `不会操作时，直接按住说话`

### 4.5 常用场景区

#### 4.5.1 推荐场景
- 挂医院号
- 查医保
- 查社保
- 交水电费
- 交话费
- 找家人帮忙

#### 4.5.2 展示方式
- 大按钮宫格或横向卡片
- 每个按钮只保留 2 到 4 个字

### 4.6 安全提醒区

#### 4.6.1 目标
- 首页就建立“先别急着操作”的心智

#### 4.6.2 默认文案
- `不认识的链接、验证码、转账要求，先别操作`
- `觉得不对劲，先联系家里人`

#### 4.6.3 跳转内容
- 防骗说明页
- 常见风险案例
- 一键联系家人

## 5. 状态设计

### 5.1 首次使用状态

#### 5.1.1 页面表现
- 展示 3 个示例问题
- 突出语音入口
- 弱化最近任务区

#### 5.1.2 引导文案
- `不会操作时，直接说给我听`
- `看不懂页面，就发截图给我`

### 5.2 已有历史用户状态

#### 5.2.1 页面表现
- 最近任务前置
- 常用任务根据历史排序
- 可展示家属是否已绑定

### 5.3 家属已绑定状态

#### 5.3.1 页面新增信息
- `已绑定 1 位家人`
- `遇到麻烦时可以直接联系`

### 5.4 未绑定家属状态

#### 5.4.1 建议文案
- `绑定家里人，遇到问题时更安心`
- 按钮：`去绑定`

---

## 6. 异常与空状态文案

### 6.1 语音识别失败
- `刚刚没听清，你再说一遍`
- 按钮：`重新说`

### 6.2 网络较差
- `现在网络有点慢，请等一下`
- 按钮：`重试`

### 6.3 截图上传失败
- `截图没有传上来，再试一次`
- 按钮：`重新上传`

### 6.4 没有找到匹配场景
- `这个问题我还没完全看懂`
- `你可以换个说法，或者发截图给我`

## 7. 首页组件优先级建议

### 7.1 MVP 必须做
- 顶部欢迎文案
- 语音主按钮
- 上传截图按钮
- 联系家人按钮
- 最近任务卡片
- 常用场景按钮
- 安全提醒卡片

### 7.2 第二阶段再做
- 动态个性化推荐
- 节日化问候
- 机构活动入口
- 运营公告位

## 8. 给设计和前端的实现建议

### 8.1 设计建议
- 首屏避免信息拥挤
- 不用复杂插画，重点是清晰可点
- 主按钮要有足够大点击面积

### 8.2 前端建议
- 首页模块顺序支持配置
- 常用场景支持后台下发
- 最近任务和家属绑定状态优先走接口驱动

### 8.3 产品建议
- 首页不要承载太多“教育用户”的内容
- 真正有效的教育在第一次成功求助之后
- 首页的核心使命只有一个：让用户马上开始求助
