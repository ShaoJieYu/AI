## 1. 用户角色定义

### 1.1 角色画像

#### 1.1.1 张老师 - 专职家教老师

```yaml
基本信息:
  年龄: 32岁
  职业: 专职家教老师
  教龄: 5年
  学生数量: 8人
  授课科目: 初中数学、高中数学

特征:
  - 以家教为主要收入来源
  - 对教学质量要求高
  - 需要管理多个学生
  - 注重效率和口碑

痛点:
  - 备课时间太长，影响收入
  - 难以针对不同学生个性化教学
  - 学生进度跟踪困难

期望:
  - 快速生成高质量备课内容
  - 系统化管理学生信息
  - 提升教学效果，获得更多推荐
```

#### 1.1.2 小李 - 兼职大学生家教

```yaml
基本信息:
  年龄: 21岁
  职业: 大三学生
  教龄: 0年（第一次做家教）
  学生数量: 2人
  授课科目: 高中物理

特征:
  - 缺乏教学经验
  - 时间有限，需要平衡学业
  - 希望获得教学指导
  - 对新技术接受度高

痛点:
  - 不知道如何备课
  - 担心教不好被家长投诉
  - 缺乏系统的教学知识

期望:
  - 获得专业的备课指导

```

## 3. 详细用户故事

### 3.1 发现阶段

#### 故事 1.1: 了解产品

```yaml
作为: 潜在用户
我想: 了解这个备课平台能做什么
以便: 判断是否符合我的需求

验收标准:
  - 可以查看产品介绍页面
  - 可以看到功能演示视频
  - 可以了解定价信息
  - 可以看到用户评价

情绪曲线:
  - 好奇 → 期待 → 信任
```

#### 故事 1.2: 体验 Demo

```yaml
作为: 潜在用户
我想: 在不注册的情况下体验产品
以便: 直观感受产品功能

验收标准:
  - 提供示例学生数据
  - 可以体验备课生成流程
  - 可以看到示例备课内容
  - 体验完整后引导注册

情绪曲线:
  - 好奇 → 惊喜 → 想要使用
```

### 3.2 注册阶段

#### 故事 2.1: 快速注册

```yaml
作为: 新用户
我想: 用最少的时间完成注册
以便: 尽快开始使用产品

验收标准:
  - 支持手机号验证码注册
  - 支持微信一键登录
  - 注册流程 < 2 分钟
  - 注册后立即进入引导

情绪曲线:
  - 期待 → 顺畅 → 兴奋
```

#### 故事 2.2: 新手引导

```yaml
作为: 新用户
我想: 了解如何使用这个产品
以便: 快速上手备课工作

验收标准:
  - 提供交互式教程
  - 展示核心功能入口
  - 提供示例数据体验
  - 可以随时跳过或回看

情绪曲线:
  - 迷茫 → 清晰 → 自信
```

### 3.3 设置阶段

#### 故事 3.1: 完善个人资料

```yaml
作为: 教师用户
我想: 完善我的教学资料
以便: 获得更精准的备课推荐

验收标准:
  - 可以填写教学科目和年级
  - 可以填写教学经验和资质
  - 可以上传头像
  - 资料完善度有提示

情绪曲线:
  - 认真 → 成就感 → 期待
```

#### 故事 3.2: 熟悉界面

```yaml
作为: 新用户
我想: 了解各个功能模块的位置
以便: 高效使用产品

验收标准:
  - 导航结构清晰易懂
  - 常用功能入口明显
  - 提供功能提示
  - 支持搜索功能

情绪曲线:
  - 探索 → 熟悉 → 掌控
```

### 3.4 备课阶段

#### 故事 4.1: 录入学生信息

```yaml
作为: 教师用户
我想: 录入学生的详细情况
以便: AI能生成针对性的备课内容

验收标准:
  - 可以录入基础信息（姓名、年级、学校）
  - 可以记录学习基础（成绩水平）
  - 可以标记薄弱知识点
  - 可以记录学习习惯和性格特点
  - 可以设定教学目标
  - 支持保存草稿

情绪曲线:
  - 认真 → 期待 → 希望

痛点:
  - 担心信息填不完整影响效果
  - 不知道哪些信息最重要

解决方案:
  - 提供必填项提示
  - 提供填写示例
  - 说明每项信息的作用
```

#### 故事 4.2: 选择备课模式

```yaml
作为: 教师用户
我想: 根据教学场景选择备课模式
以便: 生成最适合的备课内容

验收标准:
  - 提供多种备课模式（新课、复习、习题等）
  - 每种模式有说明
  - 可以设置课程时长
  - 可以调整难度系数
  - 可以添加特殊要求

情绪曲线:
  - 选择 → 确定 → 期待
```

#### 故事 4.3: AI生成备课

```yaml
作为: 教师用户
我想: 让AI自动生成备课内容
以便: 节省备课时间

验收标准:
  - 一键生成备课内容
  - 实时显示生成进度
  - 生成时间 < 10秒
  - 生成失败有提示和重试

情绪曲线:
  - 期待 → 等待 → 惊喜/失望

关键情绪点:
  - 等待时的焦虑 → 需要进度反馈
  - 看到结果时的惊喜 → 内容质量要高
```

#### 故事 4.4: 查看和编辑备课

```yaml
作为: 教师用户
我想: 查看和编辑AI生成的备课内容
以便: 根据实际需要进行调整

验收标准:
  - 内容结构清晰展示
  - 可以在线编辑所有内容
  - 支持富文本编辑
  - 支持公式编辑
  - 自动保存编辑内容
  - 支持版本管理

情绪曲线:
  - 查看 → 评估 → 调整 → 满意
```

#### 故事 4.5: 导出和打印

```yaml
作为: 教师用户
我想: 将备课内容导出或打印
以便: 在课堂上使用

验收标准:
  - 支持导出PDF
  - 支持导出Word
  - 支持直接打印
  - 导出格式美观专业

情绪曲线:
  - 完成 → 便利 → 专业
```

## 5. 痛点与机会

### 5.1 核心痛点

| 痛点 | 影响程度 | 发生阶段 | 解决方案 |
|------|----------|----------|----------|
| 备课时间太长 | 高 | 备课阶段 | AI自动生成，效率提升10倍 |
| 不知道如何个性化教学 | 高 | 备课阶段 | 基于学生画像智能推荐 |
| 缺乏教学经验 | 高 | 教学阶段 | 提供教学建议和话术 |
| 进度跟踪困难 | 中 | 跟进阶段 | 可视化进度管理 |
| 家长沟通成本高 | 中 | 跟进阶段 | 自动生成学习报告 |
| 教学资源分散 | 中 | 备课阶段 | 整合知识点库和题型库 |

### 5.2 产品机会

| 机会点 | 用户价值 | 商业价值 | 优先级 |
|--------|----------|----------|--------|
| AI智能备课 | 节省时间，提升质量 | 核心卖点 | P0 |
| 学生画像分析 | 精准教学，提升效果 | 差异化优势 | P0 |
| 多端协同 | 随时随地，无缝切换 | 提升粘性 | P1 |
| 学习报告 | 专业展示，家长满意 | 提升口碑 | P1 |
| 教学社区 | 经验分享，共同成长 | 用户活跃 | P2 |
| 智能推荐 | 个性化内容推荐 | 提升留存 | P2 |
