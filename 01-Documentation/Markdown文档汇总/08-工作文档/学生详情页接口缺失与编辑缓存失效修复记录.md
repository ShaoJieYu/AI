## 三、根本原因

### 问题一根因：前后端接口契约不对齐

前端 [StudentDetailPage.tsx](file:///d:/AI/02-Frontend/web-frontend/src/pages/Students/StudentDetailPage.tsx) 调用的 4 个接口，后端只实现了 1 个：

| 前端调用 | 后端状态 | 原返回 |
|---------|---------|--------|
| `GET /students/{id}` | 已实现 | 200 正常 |
| `GET /students/{id}/profile` | **缺失** | `code:500 No static resource` |
| `GET /students/{id}/goals` | **缺失** | `code:500 No static resource` |
| `GET /students/{studentId}/lessons` | **缺失** | `code:500 No static resource` |

### 问题二根因：React Query 缓存未失效

编辑保存的 `onSuccess` 回调只更新了 zustand store，没有失效 React Query 缓存。受影响的 queryKey 包括：

- `['students', ...]`（前缀匹配）— 学生列表、Dashboard 学生列表、备课生成页学生下拉
- `['student', id]` — 详情页主数据
- `['studentProfile', id]` — 学情档案
- `['teachingGoals', id]` — 教学目标
- `['studentLessons', id]` — 备课记录
- `['studentDashboardStats']` — 仪表盘统计聚合值

## 四、解决方案

### 修复一：后端补齐 3 个接口

**[LessonService.java](file:///d:/AI/03-Backend/backend/src/main/java/com/lessonplatform/service/LessonService.java#L91-L102)** 新增 `getLessonsByStudent(studentId)` 方法：

```java
/**
 * 查询某学生的所有备课记录（按创建时间倒序）。
 * 通过 tutorId 过滤确保只返回当前家教名下的备课，避免越权。
 */
public List<LessonPlan> getLessonsByStudent(Long studentId) {
    Long tutorId = SecurityUtils.getCurrentUserId();
    return lessonPlanMapper.selectList(new LambdaQueryWrapper<LessonPlan>()
            .eq(LessonPlan::getStudentId, studentId)
            .eq(LessonPlan::getTutorId, tutorId)
            .orderByDesc(LessonPlan::getCreatedAt));
}
```

**[StudentService.java](file:///d:/AI/03-Backend/backend/src/main/java/com/lessonplatform/service/StudentService.java#L157-L208)** 新增两个方法：

1. `getStudentProfile(id)`：从 student 表已有字段映射为前端 StudentProfile 结构
   - `weakSubjects`（"几何,函数" 逗号分隔字符串）转为数组 `["几何","函数"]`
   - `learningBasics` 映射到 `specialNeeds`
   - `personality` 文本映射到 `personality.description`
   - `knowledgeMastery` 推导 `academicLevel`（≥85 优秀 / ≥70 良好 / ≥60 中等 / 否则薄弱）
2. `getTeachingGoals(id)`：当前无 `teaching_goal` 表，返回空列表（复用 `getStudentById` 做归属校验），前端优雅地不渲染目标卡片

**[StudentController.java](file:///d:/AI/03-Backend/backend/src/main/java/com/lessonplatform/controller/StudentController.java#L56-L78)** 注入 `LessonService`，新增三个 `@GetMapping` 接口：

```java
@GetMapping("/{id}/profile")
public Result<Map<String, Object>> getStudentProfile(@PathVariable Long id) {
    return Result.success(studentService.getStudentProfile(id));
}

@GetMapping("/{id}/goals")
public Result<List<Map<String, Object>>> getTeachingGoals(@PathVariable Long id) {
    return Result.success(studentService.getTeachingGoals(id));
}

@GetMapping("/{studentId}/lessons")
public Result<List<LessonPlan>> getLessonsByStudent(@PathVariable Long studentId) {
    return Result.success(lessonService.getLessonsByStudent(studentId));
}
```

### 修复二：前端编辑保存后失效 React Query 缓存

[StudentFormPage.tsx](file:///d:/AI/02-Frontend/web-frontend/src/pages/Students/StudentFormPage.tsx#L27-L40) 引入 `useQueryClient`，新增辅助函数统一失效所有相关 queryKey：

```typescript
const queryClient = useQueryClient();

const invalidateStudentQueries = (sid: number) => {
  queryClient.invalidateQueries({ queryKey: ['students'] });
  queryClient.invalidateQueries({ queryKey: ['student', sid] });
  queryClient.invalidateQueries({ queryKey: ['studentProfile', sid] });
  queryClient.invalidateQueries({ queryKey: ['teachingGoals', sid] });
  queryClient.invalidateQueries({ queryKey: ['studentLessons', sid] });
  queryClient.invalidateQueries({ queryKey: ['studentDashboardStats'] });
};
```

在 `createMutation.onSuccess`（传新 id）和 `updateMutation.onSuccess`（传当前 id）中分别调用 `invalidateStudentQueries`。

## 八、经验总结

### 1. 前后端接口契约必须对齐

本次问题一的根因是前端调用的接口在后端不存在。这类问题的特征是：

1. 浏览器 Network 面板看到 HTTP 200（因为 Spring 把异常包装成 JSON 返回）
2. 但响应体 `code` 字段是 500
3. 前端 axios 拦截器看到 `code !== 200` 弹出 `message.error`
4. 用户感知是"页面报错"而非"接口 404"

排查时优先核对前端 API 定义文件（`api/*.ts`）与后端 Controller 的 `@GetMapping` 路径是否一一对应。

### 2. React Query mutation 后必须失效相关缓存

`useMutation` 的 `onSuccess` 回调中如果只更新 zustand store，React Query 的 `useQuery` 仍然会返回缓存数据。正确做法：

1. 列出所有受影响的 queryKey（包括前缀匹配的列表查询、详情查询、聚合统计查询）
2. 在 onSuccess 中调用 `queryClient.invalidateQueries({ queryKey: [...] })`
3. 对于按 id 查询的，要传具体的 id；对于列表/统计查询，传前缀即可

### 3. Spring 静态资源异常的特征

当 Spring 找不到匹配的 `@GetMapping` 时，会把路径当作静态资源处理，抛出 `No static resource xxx` 异常。这类异常被 `GlobalExceptionHandler` 捕获后包装成 `code:500`，前端难以区分是"接口不存在"还是"服务器内部错误"。建议在 GlobalExceptionHandler 中对 `NoResourceFoundException` 单独处理，返回 `code:404` 更准确。
