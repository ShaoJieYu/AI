## 日志收集 (ELK Stack - 规划)

### 计划架构
```
┌─────────────────────────────────────┐
│  应用日志 (后端 + AI 服务)         │
└──────────────────┬──────────────────┘
                   │
                   ▼
        ┌─────────────────────┐
        │  Logstash/Filebeat  │
        │  (日志采集)         │
        └──────────┬──────────┘
                   │
        ┌──────────▼──────────┐
        │  Elasticsearch      │
        │  (日志存储)         │
        └──────────┬──────────┘
                   │
        ┌──────────▼──────────┐
        │  Kibana             │
        │  (可视化和查询)     │
        └─────────────────────┘
```

### 实现步骤
- [ ] 部署 Elasticsearch 7.x+
- [ ] 配置 Logstash 日志采集
- [ ] 部署 Kibana 可视化
- [ ] 创建索引和仪表板
- [ ] 配置日志保留策略

### 预期收益
- 集中式日志查询
- 实时异常告警
- 性能问题分析
- 审计追踪

## 监控告警 (Prometheus + Grafana - 规划)

### 计划架构
```
┌────────────────────────────────────┐
│  应用 Metrics (Spring Boot)       │
│  /actuator/prometheus              │
└────────────┬───────────────────────┘
             │
       ┌─────▼────────┐
       │ Prometheus   │
       │ (采集存储)   │
       └─────┬────────┘
             │
       ┌─────▼────────┐
       │ Grafana      │
       │ (可视化)     │
       └──────────────┘
```

### 监控指标
- **JVM**: 堆内存、GC、线程数
- **应用**: 请求速率、响应时间、错误率
- **系统**: CPU、内存、磁盘、网络
- **业务**: 活跃用户、生成备课数、成功率

### 实现步骤
- [ ] 配置 Spring Boot Actuator
- [ ] 部署 Prometheus Server
- [ ] 部署 Grafana
- [ ] 创建自定义仪表板
- [ ] 配置告警规则

### 预期收益
- 实时性能监控
- 容量规划数据
- 异常告警通知
- 业务数据可视化
